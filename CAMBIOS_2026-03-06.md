# Cambios 2026-03-06

## Nuevo nodo: `nodo_dialogo_node`
Cerebro conversacional del robot usando TinyLlama 1.1B (Q4_K_M, inferencia CPU via llama-cpp-python).

- **Fast path (<5 ms)**: comandos conocidos ("adelante", "parar", "girar"...) devuelven respuesta canned instantanea sin tocar el LLM.
- **Slow path (LLM)**: solo para preguntas libres que no son comandos ("ana como te llamas").
- Gestiona 4 estados emocionales: `neutral`, `activo`, `atento`, `bajo`.
- Publica en `/robot_respuesta` y `/estado_emocional`.

## Wake word: "ana"
El robot solo actua tras escuchar la palabra **"ana"**. Sin ella, logea la transcripcion pero no ejecuta nada.

Flujo: `"ana adelante"` -> parser extrae `"adelante"` -> comando + respuesta instantanea.

## mic_listener_node — reescritura completa
- **Grabacion continua solapada**: graba en ping-pong (wav_A/wav_B) sin parar, incluso mientras Whisper transcribe la ventana anterior.
- **VAD por energia RMS**: solo envia a Whisper si RMS > umbral (90). Silencio = 0 CPU de inferencia.
- **Deteccion automatica de micro**: prueba `default`, `plughw:0,0`, `plughw:0,1`... y elige el primero funcional.
- **Modelo tiny + flags de velocidad**: `ggml-tiny.bin`, `-l es`, `--beam-size 1`, `--best-of 1`, `--no-flash-attn`.
- **Logea todo**: RMS de cada ventana y cada transcripcion a nivel INFO.

## object_detector_node — modo reactivo
- YOLO **dormido por defecto** (0% CPU).
- Se activa **solo por voz** ("ana mira", "ana que ves").
- Eliminado el trigger por proximidad laser.

## command_parser_node — simplificado
- Requiere wake word "ana" para procesar.
- Publica el comando **una sola vez** (eliminado el timer repetitivo de 5s).
- Si no es comando conocido, publica `pregunta:<texto>` para que el dialogo use el LLM.

## GUI eliminada del launch
`gui_dashboard_node` eliminado de `voice_controlled_turtlebot.launch.py` para liberar CPU. El nodo sigue disponible para ejecucion manual.

## Fix whisper.cpp — SIGILL
Las librerias compartidas en `install/whisper.cpp/lib/` estaban compiladas con `GGML_NATIVE=ON` (AVX) en otra maquina. Esta CPU solo tiene SSE4.2.

Solucion: recompilar con `GGML_NATIVE=OFF`, `GGML_AVX=OFF`, `GGML_F16C=OFF`, `GGML_FMA=OFF` y reinstalar en `install/`.
